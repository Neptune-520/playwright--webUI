## 定位器概述

定位器（Locator）是自动化测试中用于识别和定位网页元素的核心机制。本系统支持11种定位器类型，并提供高级功能如备用定位器和链式定位。

### 支持的定位器类型

| 类型标识 | 中文名称 | 说明 | 优先级 |
|---------|---------|------|--------|
| `css` | CSS选择器 | 最通用和推荐的定位方式 | ⭐⭐⭐⭐⭐ |
| `xpath` | XPath | 功能最强大的定位方式 | ⭐⭐⭐⭐ |
| `id` | ID | 通过元素ID定位 | ⭐⭐⭐⭐⭐ |
| `name` | Name属性 | 通过name属性定位 | ⭐⭐⭐⭐ |
| `class_name` | Class名称 | 通过CSS类名定位 | ⭐⭐⭐ |
| `tag_name` | 标签名 | 通过HTML标签定位 | ⭐⭐ |
| `text` | 文本内容 | 通过元素文本内容定位 | ⭐⭐⭐⭐ |
| `role` | ARIA角色 | 通过无障碍角色定位 | ⭐⭐⭐⭐ |
| `test_id` | Test ID | 通过data-testid属性定位 | ⭐⭐⭐⭐⭐ |
| `placeholder` | Placeholder | 通过input的placeholder定位 | ⭐⭐⭐⭐ |
| `label` | Label文本 | 通过label标签文本定位 | ⭐⭐⭐⭐ |

## 定位器数据结构

### 基础定位器结构

```json
{
  "type": "css",
  "value": ".submit-button",
  "timeout": 10000
}
```

### 完整定位器结构（含备用定位器）

```json
{
  "type": "css",
  "value": ".submit-button",
  "timeout": 10000,
  "backup_locators": [
    {
      "type": "xpath",
      "value": "//button[@type='submit']"
    },
    {
      "type": "text",
      "value": "提交"
    }
  ]
}
```

### 数据库模型中的定位器

```python
# ElementLocator 模型
{
  'name': '登录按钮',
  'code': 'login_btn',
  'locator_type': 'css',
  'locator_value': '#login-button',
  'page_url': 'https://example.com/login',
  'description': '登录页面的提交按钮',
  'wait_timeout': 10000,
  'wait_state': 'visible',
  'is_active': True
}
```

## 定位器类型详解

### 1. CSS 选择器 (css)

**最推荐的定位方式**，性能优秀，语法简洁。

#### 基础用法

```json
{
  "type": "css",
  "value": "#submit-btn"
}
```

#### 常用CSS选择器示例

##### ID选择器
```json
// 定位 id="username" 的元素
{
  "type": "css",
  "value": "#username"
}
```

##### 类选择器
```json
// 定位 class="btn-primary" 的元素
{
  "type": "css",
  "value": ".btn-primary"
}

// 定位多个类名
{
  "type": "css",
  "value": ".btn.btn-primary.btn-lg"
}
```

##### 标签选择器
```json
// 定位所有 input 元素
{
  "type": "css",
  "value": "input"
}

// 定位所有 a 标签
{
  "type": "css",
  "value": "a"
}
```

##### 属性选择器
```json
// 定位 type="text" 的 input 元素
{
  "type": "css",
  "value": "input[type='text']"
}

// 定位 name="email" 的元素
{
  "type": "css",
  "value": "[name='email']"
}

// 定位包含特定属性的元素
{
  "type": "css",
  "value": "input[required]"
}

// 定位属性值以某字符串开头的元素
{
  "type": "css",
  "value": "a[href^='https://']"
}

// 定位属性值以某字符串结尾的元素
{
  "type": "css",
  "value": "img[src$='.png']"
}

// 定位属性值包含某字符串的元素
{
  "type": "css",
  "value": "div[class*='container']"
}
```

##### 后代选择器
```json
// 定位 form 内的所有 input 元素
{
  "type": "css",
  "value": "form input"
}

// 定位 #main 内的 .item 元素
{
  "type": "css",
  "value": "#main .item"
}
```

##### 子元素选择器
```json
// 定位 ul 的直接子元素 li
{
  "type": "css",
  "value": "ul > li"
}

// 定位 table > tbody > tr
{
  "type": "css",
  "value": "table > tbody > tr"
}
```

##### 伪类选择器
```json
// 定位第一个子元素
{
  "type": "css",
  "value": "li:first-child"
}

// 定位最后一个子元素
{
  "type": "css",
  "value": "li:last-child"
}

// 定位第n个子元素
{
  "type": "css",
  "value": "tr:nth-child(2)"
}

// 定位启用的输入框
{
  "type": "css",
  "value": "input:enabled"
}

// 定位选中的复选框
{
  "type": "css",
  "value": "input[type='checkbox']:checked"
}

// 定位悬停状态的元素
{
  "type": "css",
  "value": "a:hover"
}
```

##### 复杂组合选择器
```json
// 定位导航菜单中的活动链接
{
  "type": "css",
  "value": "nav ul > li.active > a"
}

// 定位表格中的特定单元格
{
  "type": "css",
  "value": "#data-table tbody tr:nth-child(3) td:nth-child(2)"
}

// 定位特定表单中的提交按钮
{
  "type": "css",
  "value": "form#login-form button[type='submit']"
}
```

##### 特殊字符转义
```json
// ID中包含特殊字符需要转义
// HTML: <div id="user.name@example">
{
  "type": "css",
  "value": "#user\\.name\\@example"
}

// 系统会自动处理CSS ID转义
// 以下写法会被自动转义
{
  "type": "css",
  "value": "#user.name@example"
}
```

### 2. XPath (xpath)

**功能最强大的定位方式**，可以定位任何元素，支持复杂的逻辑表达式。

#### 基础用法

```json
{
  "type": "xpath",
  "value": "//button[@id='submit']"
}
```

#### 常用XPath示例

##### 绝对路径（不推荐）
```json
// 从根元素开始的完整路径
{
  "type": "xpath",
  "value": "/html/body/div[1]/form/input[2]"
}
```

##### 相对路径（推荐）
```json
// 定位页面上所有 button 元素
{
  "type": "xpath",
  "value": "//button"
}

// 定位 id="login" 的元素
{
  "type": "xpath",
  "value": "//*[@id='login']"
}

// 定位 class="active" 的元素
{
  "type": "xpath",
  "value": "//*[@class='active']"
}
```

##### 属性匹配
```json
// 定位 name="username" 的 input 元素
{
  "type": "xpath",
  "value": "//input[@name='username']"
}

// 定位 type="submit" 且 class="btn" 的元素
{
  "type": "xpath",
  "value": "//button[@type='submit' and @class='btn']"
}

// 定位 type="text" 或 type="email" 的 input
{
  "type": "xpath",
  "value": "//input[@type='text' or @type='email']"
}

// 定位没有 disabled 属性的按钮
{
  "type": "xpath",
  "value": "//button[not(@disabled)]"
}
```

##### 文本内容匹配
```json
// 定位包含文本"登录"的按钮
{
  "type": "xpath",
  "value": "//button[contains(text(), '登录')]"
}

// 定位文本完全等于"提交"的元素
{
  "type": "xpath",
  "value": "//button[text()='提交']"
}

// 定位包含特定文本的任意元素
{
  "type": "xpath",
  "value": "//*[contains(text(), '欢迎')]"
}
```

##### 位置定位
```json
// 定位第一个表格行
{
  "type": "xpath",
  "value": "//table/tr[1]"
}

// 定位最后一个列表项
{
  "type": "xpath",
  "value": "//ul/li[last()]"
}

// 定位倒数第二个元素
{
  "type": "xpath",
  "value": "//div/*[last()-1]"
}
```

##### 父子和兄弟关系
```json
// 定位父元素
{
  "type": "xpath",
  "value": "//input[@name='email']/.."
}

// 定位子元素
{
  "type": "xpath",
  "value": "//div[@class='container']/p"
}

// 定位后续兄弟元素
{
  "type": "xpath",
  "value": "//h2/following-sibling::p"
}

// 定位前面的兄弟元素
{
  "type": "xpath",
  "value": "//input/preceding-sibling::label"
}
```

##### 轴定位（复杂场景）
```json
// 从label定位其后的input
{
  "type": "xpath",
  "value": "//label[text()='用户名']/following-sibling::input"
}

// 定位包含特定文本的div内的所有span
{
  "type": "xpath",
  "value": "//div[contains(text(), '订单')]/.//span"
}

// 定位祖先元素
{
  "type": "xpath",
  "value": "//td[text()='测试']/ancestor::table"
}

// 定位所有后代元素
{
  "type": "xpath",
  "value": "//div[@id='container']//a"
}
```

##### 函数使用
```json
// starts-with 函数
{
  "type": "xpath",
  "value": "//div[starts-with(@class, 'btn-')]"
}

// normalize-space 处理空白
{
  "type": "xpath",
  "value": "//button[normalize-space(text())='登 录']"
}

// 多条件组合
{
  "type": "xpath",
  "value": "//input[@type='text' and @name='user' and not(@disabled)]"
}
```

## 高级定位器用法

### 1. 备用定位器 (Backup Locators)

当主定位器失败时，系统会自动尝试备用定位器，提高脚本的健壮性。

#### 基础示例

```json
{
  "type": "css",
  "value": "#submit-btn",
  "timeout": 10000,
  "backup_locators": [
    {
      "type": "xpath",
      "value": "//button[@type='submit']"
    }
  ]
}
```

#### 多备用定位器示例

```json
{
  "type": "css",
  "value": ".btn-primary",
  "timeout": 15000,
  "backup_locators": [
    {
      "type": "text",
      "value": "提交"
    },
    {
      "type": "xpath",
      "value": "//button[contains(@class, 'submit')]"
    },
    {
      "type": "role",
      "value": "提交"
    }
  ]
}
```

#### 实际场景：登录按钮

```json
{
  "type": "id",
  "value": "login-button",
  "timeout": 10000,
  "backup_locators": [
    {
      "type": "css",
      "value": "button[type='submit']"
    },
    {
      "type": "text",
      "value": "登录"
    },
    {
      "type": "test_id",
      "value": "login-btn"
    }
  ]
}
```

#### 实际场景：搜索框

```json
{
  "type": "placeholder",
  "value": "搜索...",
  "timeout": 8000,
  "backup_locators": [
    {
      "type": "css",
      "value": "input[name='search']"
    },
    {
      "type": "id",
      "value": "search-input"
    }
  ]
}
```

### 2. 链式定位器 (Chained Locators)

使用 ` >> ` 分隔符实现层级定位，先定位父元素，再定位子元素。

#### 基础示例

```json
{
  "type": "css",
  "value": "#container >> .item"
}
```

#### CSS + CSS 链式定位

```json
// 在表单中定位用户名输入框
{
  "type": "css",
  "value": "#login-form >> input[name='username']"
}

// 在导航中定位首页链接
{
  "type": "css",
  "value": "nav.main-nav >> a.home-link"
}

// 在表格中定位特定单元格
{
  "type": "css",
  "value": "#data-table >> tbody >> tr:first-child >> td:nth-child(2)"
}

// 在模态框中定位确认按钮
{
  "type": "css",
  "value": ".modal-dialog >> .modal-footer >> button.btn-primary"
}
```

#### CSS + 文本链式定位

```json
// 在列表中定位包含特定文本的项
{
  "type": "css",
  "value": "ul.menu >> text=首页"
}

// 在表格行中定位包含特定文本
{
  "type": "css",
  "value": "#user-table >> text=张三"
}

// 在侧边栏中定位菜单项
{
  "type": "css",
  "value": "aside.sidebar >> text=用户管理"
}
```

#### CSS + Role链式定位

```json
// 在工具栏中定位特定按钮
{
  "type": "css",
  "value": ".toolbar >> role=button"
}

// 在导航中定位菜单按钮
{
  "type": "css",
  "value": "nav >> role=button"
}
```

#### 复杂链式定位示例

```json
// 定位数据表格中第一行的操作按钮
{
  "type": "css",
  "value": "#data-table >> tbody >> tr:first-child >> .action-buttons >> button.edit-btn"
}

// 定位表单中特定字段的输入框
{
  "type": "css",
  "value": "form#user-profile >> .form-group >> label:contains('邮箱') >> input"
}

// 定位模态框中的特定文本
{
  "type": "css",
  "value": ".modal-content >> .modal-body >> text=确认删除"
}
```

### 3. 组合定位器

通过多个条件组合精确定位元素。

#### AND组合（通过backup_locators实现）

```json
{
  "type": "css",
  "value": "button",
  "backup_locators": [
    {
      "type": "css",
      "value": ".btn-primary"
    }
  ]
}
// 等价于：定位既是button又具有btn-primary类的元素
```

#### 实际应用场景

```json
// 定位可见的提交按钮
{
  "type": "tag_name",
  "value": "button",
  "backup_locators": [
    {
      "type": "text",
      "value": "提交"
    }
  ]
}

// 定位启用的输入框
{
  "type": "tag_name",
  "value": "input",
  "backup_locators": [
    {
      "type": "css",
      "value": ":not([disabled])"
    }
  ]
}
```

### 4. 等待状态和超时

#### 等待状态选项

```json
{
  "type": "css",
  "value": ".loading-indicator",
  "timeout": 10000,
  "state": "visible"
}
```

可用的等待状态：
- `visible`：元素可见（默认）
- `hidden`：元素隐藏
- `attached`：元素附加到DOM
- `detached`：元素从DOM分离

#### 超时设置示例

```json
// 快速定位（3秒超时）
{
  "type": "id",
  "value": "quick-btn",
  "timeout": 3000
}

// 标准定位（10秒超时）
{
  "type": "css",
  "value": ".standard-element",
  "timeout": 10000
}

// 慢速网络（30秒超时）
{
  "type": "xpath",
  "value": "//div[@class='slow-loading']",
  "timeout": 30000
}
```

## 最佳实践

### 1. 定位器优先级选择

**推荐顺序**（从高到低）：

1. **test_id** - 最稳定，专为测试设计
2. **id** - 快速且唯一
3. **css** - 灵活且性能优秀
4. **placeholder** - 表单输入框专用
5. **label** - 语义化的表单定位
6. **text** - 直观但可能变化
7. **role** - 无障碍标准
8. **name** - 表单元素传统方式
9. **xpath** - 功能强大但性能稍慢
10. **class_name** - 可能不唯一
11. **tag_name** - 最不精确

### 2. 定位器命名规范

```json
// 好的命名
{
  "type": "test_id",
  "value": "login-submit-button"
}

{
  "type": "css",
  "value": "input[name='user-email']"
}

// 避免的命名
{
  "type": "css",
  "value": "#btn1"  // 不明确
}

{
  "type": "css",
  "value": "div > div > span"  // 脆弱的路径
}
```

### 3. 使用备用定位器提高稳定性

```json
{
  "type": "test_id",
  "value": "submit-btn",
  "backup_locators": [
    {
      "type": "css",
      "value": "button[type='submit']"
    },
    {
      "type": "text",
      "value": "提交"
    }
  ]
}
```

### 4. 避免绝对XPath

```json
// 不推荐：绝对路径，容易因页面结构变化而失效
{
  "type": "xpath",
  "value": "/html/body/div[1]/form/div[2]/input"
}

// 推荐：相对路径，使用属性定位
{
  "type": "xpath",
  "value": "//form[@id='login']//input[@name='password']"
}
```

### 5. 合理设置超时时间

```json
// 快速加载的元素
{
  "type": "id",
  "value": "header-logo",
  "timeout": 5000
}

// 需要等待加载的动态内容
{
  "type": "css",
  "value": ".async-data-table",
  "timeout": 30000
}

// 使用默认超时（10秒）
{
  "type": "css",
  "value": ".standard-element"
}
```

### 6. 使用链式定位器提高可读性

```json
// 不推荐：复杂的单一选择器
{
  "type": "css",
  "value": "#main .content form#user-profile .form-group:nth-child(3) input"
}

// 推荐：使用链式定位器
{
  "type": "css",
  "value": "#user-profile >> .form-group:nth-child(3) >> input"
}
```

### 7. 处理动态内容

```json
// 使用contains处理动态类名
{
  "type": "css",
  "value": "div[class*='loading-']"
}

// 使用XPath处理动态ID
{
  "type": "xpath",
  "value": "//div[starts-with(@id, 'user-')]"
}

// 使用属性前缀匹配
{
  "type": "css",
  "value": "div[id^='item-']"
}
```

### 8. 处理Iframe中的元素

```json
// 先定位iframe，再定位内部元素
{
  "type": "css",
  "value": "iframe#editor >> css=body >> p:first-child"
}
```

## 常见问题排查

### 问题1：元素找不到

**可能原因**：
1. 定位器值错误
2. 元素还未加载
3. 元素在iframe中
4. 元素被隐藏

**解决方案**：

```json
// 1. 增加超时时间
{
  "type": "css",
  "value": ".dynamic-element",
  "timeout": 30000
}

// 2. 使用备用定位器
{
  "type": "css",
  "value": ".primary-selector",
  "backup_locators": [
    {
      "type": "xpath",
      "value": "//alternative/path"
    }
  ]
}

// 3. 等待元素可见状态
{
  "type": "css",
  "value": ".hidden-element",
  "state": "visible"
}
```

### 问题2：找到多个元素

**解决方案**：

```json
// 使用更精确的定位器
{
  "type": "css",
  "value": "#unique-id"
}

// 使用nth选择器指定第几个
{
  "type": "css",
  "value": ".items:nth-child(2)"
}

// 使用链式定位器缩小范围
{
  "type": "css",
  "value": "#parent-container >> .target-item"
}
```

### 问题3：点击被阻挡

**解决方案**：

```json
// 使用force点击
{
  "type": "css",
  "value": ".overlay-button",
  "action_config": {
    "force": true
  }
}

// 使用XPath找到真正的可点击元素
{
  "type": "xpath",
  "value": "//button[@type='submit' and not(@disabled)]"
}
```

### 问题4：元素定位不稳定

**解决方案**：

```json
// 使用多个备用定位器
{
  "type": "test_id",
  "value": "submit-button",
  "timeout": 15000,
  "backup_locators": [
    {
      "type": "css",
      "value": "button[type='submit']"
    },
    {
      "type": "text",
      "value": "提交"
    },
    {
      "type": "role",
      "value": "提交"
    }
  ]
}

// 使用更稳定的属性
{
  "type": "css",
  "value": "input[name='email']"  // 比class更稳定
}
```
